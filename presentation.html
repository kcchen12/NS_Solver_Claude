What Problem Does It Solve?

Uniform flow in a 2-D rectangular channel at low-to-moderate Reynolds numbers, with an optional circular cylinder obstacle producing a Kármán vortex street.

∂u/∂t + (u·∇)u = −∇p + ν ∇²u + fIBM ∇·u = 0 (incompressibility constraint)

Domain

Rectangular box
Default: 4.0 × 2.0 units

Boundary faces

Left: inflow (u = U∞)
Right: convective outflow
Top/Bottom: no-slip walls

Reynolds number

Re = U∞·D / ν
Default: Re = 200
Kinematic viscosity ν = 1/Re

src/grid.py — MAC Staggered Grid

The Marker-And-Cell (MAC) grid places variables at different physical locations, preventing checkerboard pressure oscillations.

Variable Location Shape ──────── ───────────── ───────────── p cell centres (nx, ny) u x-face centres (nx+1, ny) v y-face centres (nx, ny+1) Example (nx=4, ny=3): y ↑ ┌──●──┬──●──┬──●──┬──●──┐ │ │ │ │ │ u→ face │ p │ p │ p │ │ │ │ │ │ ├──●──┼──●──┼──●──┼──●──┤ v↑ faces cross row boundaries └──●──┴──●──┴──●──┴──●──┘→ x

3-D support included: set nz>1 to activate z-faces for w

Key attributes

  • nx, ny — number of cells
  • dx, dy — uniform cell spacings
  • xc, yc — cell-centre coordinates
  • xf, yf — face coordinates (nx+1 pts)
  • zeros_u/v/p() — zero-filled array factories
  • is_3d — 2-D / 3-D mode flag

Why staggering matters

Velocity faces share indices with neighbouring pressure cells, so discrete divergence is computed without interpolation — no checkerboard pressure instability possible.

div[i,j] = (u[i+1,j]−u[i,j])/dx + (v[i,j+1]−v[i,j])/dy

src/grid.py — CartesianGrid class

class CartesianGrid:
    """Uniform Cartesian grid supporting 2-D and 3-D problems."""
    def __init__(self, nx, ny, nz=1, lx=1.0, ly=1.0, lz=1.0):
        self.nx, self.ny, self.nz = nx, ny, nz
        self.dx = lx / nx          # uniform cell width
        self.dy = ly / ny          # uniform cell height

        # Face (node) coordinates — nx+1 points spans [0, lx]
        self.xf = np.linspace(0.0, lx, nx + 1)
        self.yf = np.linspace(0.0, ly, ny + 1)

        # Cell-centre coordinates — midpoint of each face pair
        self.xc = 0.5 * (self.xf[:-1] + self.xf[1:])
        self.yc = 0.5 * (self.yf[:-1] + self.yf[1:])

        self.is_3d = (nz > 1)

    @property
    def u_shape(self): return (self.nx + 1, self.ny)   # x-faces
    @property
    def v_shape(self): return (self.nx, self.ny + 1)   # y-faces
    @property
    def p_shape(self): return (self.nx, self.ny)         # cell centres

    def zeros_u(self): return np.zeros(self.u_shape)
    def zeros_v(self): return np.zeros(self.v_shape)
    def zeros_p(self): return np.zeros(self.p_shape)

    def __repr__(self):
        dim = "3D" if self.is_3d else "2D"
        return f"CartesianGrid({dim}, nx={self.nx}, ny={self.ny}, dx={self.dx:.4g})"

Divergence formula

div[i,j] = (u[i+1,j]−u[i,j])/dx + (v[i,j+1]−v[i,j])/dy

Convection — flux form

Written as −d(u²)/dx − d(uv)/dy to conserve momentum exactly on the discrete grid

src/operators.py — Key Implementations

def divergence(u, v, grid):
    """∇·u at cell centres — shape (nx, ny). Should be ≈ 0 after projection."""
    return ((u[1:,:] - u[:-1,:]) / grid.dx
          + (v[:,1:] - v[:,:-1]) / grid.dy)

def laplacian_u(u, grid):
    """∇²u at interior x-faces — 5-point centred stencil."""
    dx2, dy2 = grid.dx**2, grid.dy**2
    ui = u[1:-1, :]    # interior x-faces
    return ((u[2:,:]  - 2*ui + u[:-2,:]) / dx2
          + (u[1:-1,1:] - 2*ui + u[1:-1,:-1]) / dy2)

def convection_u(u, v, grid):
    """−(u·∇)u in conservative flux-divergence form at interior x-faces."""
    dx, dy = grid.dx, grid.dy
    # Interpolate v to u-face corners (average 4 surrounding v-faces)
    v_at_uf = 0.25*(v[:-1,:-1] + v[:-1,1:] + v[1:,:-1] + v[1:,1:])
    # d(u²)/dx : right and left face-averaged u
    uu_r = ((0.5*(u[1:-1,:] + u[2:,:]))**2)
    uu_l = ((0.5*(u[1:-1,:] + u[:-2,:]))**2)
    # d(uv)/dy : top and bottom face-averaged uv product
    uv_t = (0.5*(u[1:-1,1:] + u[1:-1,:-1])) * v_at_uf[:,1:]
    uv_b = (0.5*(u[1:-1,1:] + u[1:-1,:-1])) * v_at_uf[:,:-1]
    return -(uu_r - uu_l)/dx - (uv_t - uv_b)/dy

src/boundary.py — Boundary Conditions

BC types

  • INFLOW — prescribe u = U∞, v = 0
  • OUTFLOW — convective: ∂φ/∂t + c ∂φ/∂x = 0
  • WALL — no-slip (u=v=0) via ghost cells
  • FARFIELD — free-stream velocity
  • PERIODIC — handled by MPI ghost exchange

Ghost-cell technique

Values at halo rows/columns are set so interior stencils automatically see the correct BC — no special-casing inside operators.

No-slip top wall (y = ly): u_ghost[top] = −u[interior] → average at wall = 0 ✓ Convective outflow (right, x = lx): φ_new = φ_old − c·dt·(φ_old−φ_prev)/dx

src/rk3.py — SSP-RK3 Time Integrator

Shu-Osher (1988) three-stage scheme — 3rd-order accurate and Strong Stability Preserving (TVD property maintained for convection).

u¹ = uⁿ + Δt·L(uⁿ) u² = ¾ uⁿ + ¼(u¹ + Δt·L(u¹)) uⁿ⁺¹= ⅓ uⁿ + ⅔(u² + Δt·L(u²))

L(u,v) = −(u·∇)u + ν∇²u
(convection + diffusion, without pressure)


Why SSP?

For advection-dominated flows the SSP property prevents spurious oscillations that could destabilise the simulation. The .copy() calls are inherent to SSP — unavoidable, not a bug.

def ssp_rk3_step(rhs_fn, u, v, dt):
    # ── Stage 1 ──────────────────────────
    Lu1, Lv1 = rhs_fn(u, v)
    u1 = u.copy(); v1 = v.copy()
    u1[1:-1,:] = u[1:-1,:] + dt * Lu1
    v1[:,1:-1] = v[:,1:-1] + dt * Lv1

    # ── Stage 2 ──────────────────────────
    Lu2, Lv2 = rhs_fn(u1, v1)
    u2 = u.copy(); v2 = v.copy()
    u2[1:-1,:] = (0.75*u[1:-1,:]
                + 0.25*(u1[1:-1,:] + dt*Lu2))
    v2[:,1:-1] = (0.75*v[:,1:-1]
                + 0.25*(v1[:,1:-1] + dt*Lv2))

    # ── Stage 3 ──────────────────────────
    Lu3, Lv3 = rhs_fn(u2, v2)
    u_new = u.copy(); v_new = v.copy()
    u_new[1:-1,:] = ((1/3)*u[1:-1,:]
                   + (2/3)*(u2[1:-1,:] + dt*Lu3))
    v_new[:,1:-1] = ((1/3)*v[:,1:-1]
                   + (2/3)*(v2[:,1:-1] + dt*Lv3))
    return u_new, v_new

src/solver.py — Fractional-Step Solver

Implements the Kim & Moin (1985) projection method. Each call to solver.step(dt) performs five sub-steps:

① SSP-RK3
no pressure
→
② Poisson
∇²φ = ∇·u*/dt
→
③ Correct u
u = u*−dt∇φ
→
④ Update p
p += φ
→
⑤ Apply BCs
+ IBM

Step 1 — Advection-diffusion

Three RK stages; BCs and IBM applied at every stage to intermediate fields to maintain stability.

Step 2 — Pressure Poisson

Enforces ∇·u = 0 to machine precision. Sparse direct solver (matrix cached from init).

Steps 3–5 — Projection

Subtract pressure gradient from u*; accumulate φ into p; reapply essential BCs only.


Adaptive time step

suggest_dt(): min of convective CFL limit, diffusion limit, and dt_max.

CFL diagnostic

cfl(dt): max Courant number over all cells. Target: CFL ≤ 0.3

Divergence check

divergence(): should be ~1e-14 after projection. Confirms mass conservation.

src/solver.py — step(dt) code

def step(self, dt):
    # ── 1. SSP-RK3: convection+diffusion, no pressure ──────────────
    def _rhs(u, v):
        apply_velocity_bc(u, v, self.grid, self.bc, dt=dt)
        if self.ibm and self.ibm.has_solid: self.ibm.apply(u, v)
        return rhs_u(u, v, ...), rhs_v(u, v, ...)

    u_star, v_star = ssp_rk3_step(_rhs, self.u, self.v, dt)
    apply_velocity_bc(u_star, v_star, ...)   # final BCs on u*
    if self.ibm: self.ibm.apply(u_star, v_star)

    # ── 2. Solve pressure Poisson: ∇²φ = ∇·u* / dt ─────────────────
    div_star = divergence(u_star, v_star, self.grid)
    phi = self._poisson.solve(div_star / dt)
    apply_pressure_bc(phi, ...)

    # ── 3. Velocity correction ──────────────────────────────────────
    u_new = u_star.copy()
    u_new[1:-1,:] -= dt * (phi[1:,:] - phi[:-1,:]) / grid.dx  # x-faces
    v_new = v_star.copy()
    v_new[:,1:-1] -= dt * (phi[:,1:] - phi[:,:-1]) / grid.dy  # y-faces

    # ── 4. Pressure update ──────────────────────────────────────────
    self.p += phi

    # ── 5. Post-correction BCs (essential only) ─────────────────────
    apply_post_correction_bc(u_new, v_new, ...)
    if self.ibm: self.ibm.apply(u_new, v_new)
    self.u, self.v = u_new, v_new
    self.t += dt

src/poisson.py — Pressure Poisson Solver

Solves ∇²φ = ∇·u* / Δt on the cell-centred grid using a sparse direct method.

5-point stencil

φ[i,j+1] | φ[i-1,j]── φ[i,j] ──φ[i+1,j] | φ[i,j-1] (φ[i+1,j]−2φ[i,j]+φ[i-1,j])/dx² +(φ[i,j+1]−2φ[i,j]+φ[i,j-1])/dy² = rhs[i,j]

Boundary conditions

  • Neumann ∂φ/∂n=0 at walls/inflow: ghost-cell coefficient folded into diagonal
  • Dirichlet φ=0 at outflow: entire row replaced by 1·φ=0
  • Regularisation: for purely Neumann systems pin φ[0,0]=0 (remove null space)

Performance: matrix cached

class PoissonSolver:
    def __init__(self, grid, bc):
        # Build sparse CSR matrix ONCE
        self._A, self._dmask = build_poisson_matrix(grid, bc)

    def solve(self, rhs):
        b = rhs.flatten()
        b[self._dmask] = 0.0    # enforce Dirichlet in RHS
        phi = spsolve(self._A, b)
        return phi.reshape((nx, ny))

Why direct solver?

For grid sizes up to ~512×256 the direct sparse factorisation is faster and more robust than iterative methods (no iteration count tuning needed).

src/ibm.py — Immersed Boundary Method

Solid objects modelled inside the fluid without re-meshing — represented by Boolean masks on the MAC grid.

Direct-Forcing Algorithm

  1. Compute intermediate velocity u* normally
  2. Identify MAC faces inside solid geometry
  3. Force those face velocities to ubody = 0
u*[mask_u] = 0 v*[mask_v] = 0

Equivalent to body force f = (ubody−u*)/Δt in momentum equation


Geometry builders

  • add_circle(cx, cy, r) — circular cylinder
  • add_rectangle(x0,x1,y0,y1) — axis-aligned box
  • add_mask(mask_u, mask_v) — custom boolean arrays
class ImmersedBoundary:
    def __init__(self, grid):
        self.mask_u = np.zeros(grid.u_shape, dtype=bool)
        self.mask_v = np.zeros(grid.v_shape, dtype=bool)

    def add_circle(self, cx, cy, radius):
        # Mark u-faces inside circle
        xf2d, yc2d = np.meshgrid(grid.xf, grid.yc, indexing='ij')
        dist_u = np.sqrt((xf2d-cx)**2 + (yc2d-cy)**2)
        self.mask_u |= (dist_u <= radius)
        # Similar for v-faces using (xc, yf)

    def apply(self, u, v, u_body=0.0, v_body=0.0):
        """Direct forcing: overwrite solid face velocities."""
        u[self.mask_u] = u_body   # ← the entire IBM in 2 lines
        v[self.mask_v] = v_body

    @property
    def has_solid(self):
        return bool(self.mask_u.any() or self.mask_v.any())

Called every RK sub-stage

Intermediate u* is also forced, so the no-penetration condition is maintained throughout the Runge-Kutta integration.

src/parallel.py — MPI Domain Decomposition

Strategy: 1-D y-slabs

Rank 0 ┌──────────────┐ j = ny−1 │ ny_local │ │ rows + ghosts│← ghost from rank 1 Rank 1 ├──────────────┤ │ ny_local │ │ rows + ghosts│ Rank 2 ├──────────────┤ │ ny_local │ j = 0 └──────────────┘ Each rank: ny_global / nproc rows + 1 ghost cell on each side for the 5-point stencil.

Key methods

  • exchange_ghost_u/v() — MPI_Sendrecv between neighbouring ranks (halo update)
  • gather_to_root() — collect local arrays to rank 0 for Poisson solve
  • scatter_from_root() — distribute pressure correction φ back to all ranks

Serial fallback

try:
    from mpi4py import MPI
    comm = MPI.COMM_WORLD
except ImportError:
    comm = None   # single-rank serial mode

Limitation

Poisson gathered to rank 0 and solved serially — limits scalability beyond ~8–16 MPI ranks. Future work: distributed multigrid Poisson.
